# Martin Vinyl's Record Store — 技术架构

## 技术栈总览

| 层级 | 技术 | 说明 |
|------|------|------|
| 结构层 | HTML5 | 3 个独立 HTML 页面，无 SPA 框架 |
| 样式层 | CSS3 | 单文件 `styles.css`，无预处理器 |
| 逻辑层 | Vanilla JavaScript (ES6+) | 无框架，无构建工具 |
| AI 服务 | OpenAI GPT-4 API | NPC 对话、风格推荐、情绪分析 |
| 音频渲染 | HTML5 Audio API | 音频播放与状态管理 |
| 图形渲染 | Canvas 2D API | 波形动画可视化 |
| 状态持久化 | Web Storage API (localStorage) | 跨页面状态共享 |
| 字体 | 自定义 TTF 字体（IPix） | 像素风字体，本地加载 |

## 模块职责划分

### `script.js` — 对话页主控制器

负责对话页的全部交互逻辑，是应用最核心的文件。

```
主要职责：
├── NPC System Prompt 定义（角色设定、对话规则）
├── 聊天消息收发（sendMessage / appendMessage）
├── GPT-4 API 调用（generateNPCResponse）
├── 标签解析与渲染（\mood \style{} \story{} → 橙色 span）
├── 标签收集面板管理（selectText / checkWordsAndShowTip）
├── 推荐触发逻辑（recommendMusicStyle / recommendSong）
├── NPC 状态管理（背景图切换 + 浮动动画控制）
├── 背景视差效果（handleMouseMove）
├── localStorage 读写（聊天历史 / NPC 状态 / 推荐结果）
└── 背景音乐播放（自动播放 + 用户交互解锁）
```

**关键函数调用链：**

```
用户输入 Enter
  └─ sendMessage()
       ├─ appendMessage('我', message)        # 渲染用户消息
       ├─ generateNPCResponse(message)        # 调用 GPT-4
       │    └─ fetch OpenAI API
       │         └─ 解析 \mood{} \style{} \story{} 标记
       └─ appendMessage('店主', npcResponse)  # 渲染 NPC 回复（含可点击标签）

用户点击橙色标签词
  └─ selectText(type, text)
       └─ checkWordsAndShowTip()
            ├─ [条件满足] recommendMusicStyle()  # 风格推荐 API 调用
            ├─ recommendSong(songs)              # 情绪分析 + 歌曲匹配
            └─ 显示 recommendTip 提示框
```

---

### `player.js` — 播放器控制器

```
主要职责：
├── 曲目数据定义（tracks 数组，含 id/image/audio）
├── 曲目播放控制（playTrack）
│    ├─ 更新 active 样式
│    ├─ 切换唱片封面叠加图（playerOverlay.src）
│    └─ 控制 audioPlayer 播放
├── Canvas 波形动画（drawWave）
│    ├─ 3 条叠加正弦波，各有独立频率/振幅/速度/颜色
│    ├─ 播放时：振幅随机跳动（20~60px），动画速度 ×3
│    └─ 暂停时：振幅趋近 0，动画速度 ×0.1
└─ 音频事件监听（play/pause → 波形状态切换）
```

**波形参数设计：**

```javascript
waves = [
  { frequency: 0.02, amplitude: 20, speed: 0.05, color: 'rgba(255,152,0,0.9)' },
  { frequency: 0.03, amplitude: 30, speed: 0.07, color: 'rgba(255,152,0,0.8)' },
  { frequency: 0.04, amplitude: 40, speed: 0.09, color: 'rgba(255,152,0,0.7)' }
]
```

### `audio.js` — 跨页面音频管理器

```
主要职责：
├── 根据当前页面路径选择音频源
│    ├─ output.html → final songs/ 目录下的推荐歌曲
│    └─ 其他页面   → 背景音乐 Echoes of Vinyl
├── 音量设置（0.3，低音量背景播放）
├── 自动播放处理（Promise + 用户交互解锁）
└── 错误诊断日志（networkState / readyState）
```

---

### `songRecommendation.js` — AI 推荐引擎

独立模块，使用 ES Module `export` 导出三个函数（注：当前项目未使用 `<script type="module">`，存在加载方式不一致的问题）。

```
recommendSong(songs, moodWords, styleWords)
  ├─ recommendMusicStyle(moodWords, styleWords)
  │    └─ GPT-4：从 7 种风格中选最匹配的一种
  ├─ analyzeMoodAndIntensity(moodWords, styleWords)
  │    └─ GPT-4：返回 [A-E][M-Q] 两位情绪编码
  └─ 歌曲匹配逻辑
       ├─ 精确匹配：情绪编码 + 风格编码 完全一致
       ├─ 降级匹配：仅风格编码一致，随机选取
       └─ 兜底：返回 'Echoes of Vinyl_CM02.mp3'
```

## AI 集成架构

### API 调用模式

所有 AI 调用均为客户端直接请求，无后端代理：

```
Browser → fetch('https://api.openai.com/v1/chat/completions')
            ├─ model: "gpt-4"
            ├─ temperature: 0.7
            └─ Authorization: Bearer [硬编码 API Key]  ⚠️ 安全风险
```

### 三次 AI 调用时序

```
对话阶段（每轮对话触发一次）：
  generateNPCResponse(userText)
    → System Prompt: 马丁·黑胶角色设定
    → 返回：含 \mood{} \style{} \story{} 标记的对话文本

推荐阶段（标签积累到阈值后触发，共两次并行意图，实际串行）：
  recommendMusicStyle(moodWords, styleWords)
    → System Prompt: 音乐风格推荐专家
    → 返回：单一风格名称字符串

  analyzeMoodAndIntensity(moodWords, styleWords)
    → System Prompt: 音乐情绪分析专家
    → 返回：两位字母编码（如 "BN"）
```

## 状态管理架构

项目无状态管理库，采用 **DOM + localStorage** 双层状态模型：

```
运行时状态（内存）          持久化状态（localStorage）
─────────────────          ──────────────────────────
dialogueCount              chatHistory（HTML 字符串）
hasShownTip                npcState（背景图 URL）
hasShownPlayerTip          currentTrack（曲目 ID）
recommendedSongPath        recommendedSong（歌曲路径）
isPlaying（播放器）         hasShownRecommendTip
```

**跨页面通信方式：** 通过 `localStorage` 传递推荐结果，无 URL 参数或 postMessage。

## 音频系统架构

```

  └─ <audio id="backgroundAudio" loop>
       ├─ 音量：0.3
       ├─ 自动播放（受浏览器策略限制，首次点击解锁）
       └─ audio.js 统一管理

播放器音频层（player.html）
  └─ <audio id="audioPlayer">
       ├─ 音量：默认（1.0）
       ├─ 手动触发（点击曲目）
       └─ player.js 管理，事件驱动波形动画
```

## Canvas 渲染管线

```
初始化：
  canvas.width  = window.innerWidth × 0.75
  canvas.height = 600px

每帧（requestAnimationFrame）：
  ctx.clearRect()
  for each wave:
    ├─ 更新 amplitude（播放中随机扰动 / 暂停时衰减）
    ├─ 绘制正弦曲线（sin(x × frequency + offset) × amplitude）
    ├─ strokeStyle = rgba(255, 152, 0, α)  // 橙色，各波透明度递减
    └─ offset += speed × animationSpeed   // 推进相位，产生流动感
```
