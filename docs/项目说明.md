# Martin Vinyl's Record Store — 项目说明

## 项目概述

**Martin Vinyl's Record Store** 是一个以复古黑胶唱片店为主题的交互式音乐推荐 Web 应用。用户通过与 NPC 角色"马丁·黑胶"进行自然语言对话，倾诉自己的心情与故事，由 AI 分析情绪、风格偏好，最终为用户推荐一首最契合当下心境的音乐。

整个体验模拟了"走进一家老唱片店，向店主诉说心事，由他为你挑选一张唱片"的沉浸式叙事流程。

---

## 核心体验流程

```
用户进入对话页 → 与 NPC 聊天 → AI 提取情绪/风格/故事标签
       ↓
标签积累到阈值 → AI 推荐音乐风格 + 情绪编码
       ↓
跳转播放器页 → 试听各风格样带 → 查看礼品包装台（推荐结果展示）
```

---

## 页面结构

### 1. 对话页 `index.html` — 复古唱片店主厅

这是应用的核心入口页面，承载了主要的交互逻辑。

**界面元素：**
- 视差背景：鼠标移动时背景图轻微位移，营造空间纵深感
- NPC 角色（马丁·黑胶）：左侧浮动的像素风人物，根据对话状态切换表情（睡眠/普通/开心/悲伤/可爱）
- 对话框：右下角半透明暗色聊天窗口，支持历史记录滚动
- 三栏标签面板（顶部）：实时展示 AI 从对话中提取的 **心情 / 风格 / 故事** 关键词
- 推荐提示框：当三类标签各积累 ≥2 个时，右侧弹出橙色提示卡，告知推荐结果已生成

**交互逻辑：**
- 用户在输入框输入文字，按 Enter 发送
- NPC 调用 GPT-4 生成回复，回复中的情绪/风格/故事词汇以橙色高亮显示，可点击收集
- 对话进行 4 轮后，NPC 会主动提示用户去试听播放器
- 对话历史通过 `localStorage` 持久化，刷新页面不丢失

---

### 2. 播放器页 `player.html` — 样带试听室

供用户在推荐前了解各音乐风格的试听页面。

**界面元素：**
- 左侧曲目列表：7 种音乐风格，悬停显示播放按钮，选中项高亮为橙色
- 右侧唱片机：像素风唱片机底座 + 动态封面叠加层（随选曲切换）
- Canvas 波形动画：右侧全屏波浪，播放时振幅随机跳动，暂停时趋近平静

### 3. 礼品包装台 `output.html` — 推荐结果展示

展示最终推荐唱片的"礼品包装"页面，模拟店主将唱片包装好递给你的仪式感。

**界面元素：**
- 居中浮动的唱片封面图（带上下浮动动画）
- 底部文字区：唱片名称、店铺名、创作信息
- 左下角：音乐风格标签 + 实时时间（每秒更新）
- 背景：专属的物品陈列背景图

## NPC 角色设定

**马丁·黑胶（Martin Vinyl）**

- 年龄：55 岁
- 性格：郁郁不得志，内心深藏对复古音乐的执着与哀愁，看似犀利实则充满无奈
- 对话风格：简短（≤90字），略带讽刺的社会评论，针对性音乐点评，以反问收尾
- 不推荐现实中存在的音乐作品（全部为虚构曲目）
- 对话终止条件：用户确认情绪/风格/故事后，NPC 说"好的，我在为你寻找合适的唱片，请稍等……"

**NPC 状态图：**
```
npc-sleep.png   → 默认待机状态
npc-normal.png  → 对话中（普通）
npc-happy.png   → 用户情绪积极时
npc-sad.png     → 用户情绪消极时
npc-cute.png    → 特殊互动状态
```

## AI 推荐系统

### 标签提取

NPC 回复中使用特殊标记语法，前端解析后渲染为可点击的橙色高亮词：

```
\mood{情绪词}   → 心情标签（如：失落、平静、愤怒）
\style{风格词}  → 风格标签（如：忧郁、激烈、温柔）
\story{故事词}  → 故事标签（如：失恋、离职、旅行）
```

### 推荐触发条件

三类标签各 ≥2 个时，自动触发推荐流程：

1. **风格推荐**：将情绪词 + 风格词发送给 GPT-4，从 7 种预设风格中选出最匹配的一种
2. **情绪编码**：GPT-4 分析情绪词，返回两位编码（情绪级别 A-E × 激烈程度 M-Q）
3. **歌曲匹配**：按 `[情绪编码][风格编码].mp3` 命名规则从歌曲库中精确匹配，无精确匹配则降级为同风格随机选取

### 歌曲命名规范

```
[曲名]_[情绪级别][激烈程度][风格编码].mp3

情绪级别：A（最积极）→ E（最消极）
激烈程度：M（最平静）→ Q（最激荡）
风格编码：01=Barbershop, 02=Baroque, 03=Blues,
          04=Big Band Jazz, 05=Funk&RnB, 06=Hardcore Punk, 07=Folk

示例：Silent Whisper of the Pines_BM07.mp3
      → 情绪偏消极(B)，节奏平静(M)，Folk风格(07)
```

## 数据持久化

使用 `localStorage` 跨页面共享状态：

| Key | 内容 |
|-----|------|
| `chatHistory` | 聊天记录 HTML 片段 |
| `npcState` | NPC 当前背景图 URL |
| `currentTrack` | 播放器当前曲目 ID |
| `recommendedSong` | 推荐歌曲文件路径 |
| `hasShownRecommendTip` | 推荐提示是否已显示 |

## 已知问题与注意事项

1. **API Key 暴露**：OpenAI API Key 硬编码在客户端 JS 文件中（`script.js`、`songRecommendation.js`），存在严重安全风险，生产环境必须迁移至后端代理
2. **无构建工具**：纯静态文件，需通过本地服务器（如 Live Server）运行，直接双击 HTML 文件会因跨域限制导致音频/字体加载失败
3. **桌面端优化**：CSS 使用固定定位和 vw/vh 单位，无媒体查询，移动端体验未经优化
4. **歌曲库未完整**：`script.js` 中的歌曲库数组仅有占位符，实际歌曲文件存放在 `final songs/` 目录（未包含在仓库中）
