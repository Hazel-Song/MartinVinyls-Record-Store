# 功能亮点详解

## 1. AI 情感理解系统

### 不是"你喜欢什么"，而是"你现在怎么样"

传统音乐推荐的逻辑是：
- 你听过什么 → 推荐相似的
- 你喜欢什么风格 → 推荐这个风格的热门

**Martin Vinyl's Record Store** 的逻辑是：
- 你现在的情绪状态 → 匹配对应情绪的音乐
- 你说不清楚的感觉 → AI 帮你提炼成标签

### 三维情感坐标系

AI 会从对话中提取三类信息：

**心情（Mood）**
- 失落、平静、愤怒、兴奋、疲惫……
- 不是简单的"开心/难过"二分法，而是细腻的情绪光谱

**风格（Style）**
- 忧郁、激烈、温柔、躁动、空灵……
- 描述的是你想要的音乐"质感"，而非音乐类型

**故事（Story）**
- 失恋、离职、旅行、深夜、雨天……
- 情绪发生的场景和背景，帮助 AI 理解上下文

当三类标签各积累到 2 个以上，AI 会综合分析，生成一个二维编码：
- **情绪级别**（A-E）：从积极到消极
- **激烈程度**（M-Q）：从平静到激荡

然后从歌曲库中匹配最契合的那一首。

### 智能对话引导

马丁不会直接问你"你现在心情怎么样"——那太生硬了。他会：
- 根据你的话自然地反问
- 在回复中嵌入情绪词汇，等你点击
- 在合适的时机总结："看起来你经过了【故事】后，现在感觉【心情】，想要一首【风格】的音乐来缓解心情。对吗？"

这种引导是隐形的，你感觉不到在"填表"，只是在聊天。

## 2. 活的 NPC 角色

### 马丁·黑胶的人设

**年龄**：55 岁  
**职业**：复古唱片店主  
**性格**：郁郁不得志，内心深藏对复古音乐的执着与哀愁  
**说话风格**：简短（≤90字）、略带讽刺、针对性音乐点评、以反问收尾

**典型对话示例：**

> 用户："今天加班到很晚，累死了"  
> 马丁："加班？呵，现代人的通病。你是想要\style{平静}的音乐放松一下，还是\style{激烈}的节奏发泄一下？"

> 用户："分手了，心情很糟"  
> 马丁："又一个被爱情伤透的灵魂。\mood{悲伤}的时候，有人想要更悲伤的音乐宣泄，有人想要\style{温柔}的旋律疗伤。你呢？"

### 表情系统

马丁有 5 种状态，会根据对话内容动态切换：

- **睡眠**（默认）：等待用户唤醒
- **普通**：对话进行中
- **开心**：用户情绪积极时
- **悲伤**：用户情绪消极时
- **可爱**：特殊互动状态

表情切换通过 CSS `background-image` 实现，0.3s 平滑过渡，配合浮动动画，让 NPC 真的"活"起来。

### 不推荐现实作品

马丁的货架上全是虚构曲目：
- *Echoes of Vinyl*（背景音乐）
- *Burning Pages*（Barbershop）
- *Ghosts of Yesterday*（Blues）
- *Shattered Chains*（Hardcore Punk）
- ……

这个设定有两个好处：
1. **避免版权问题**：不涉及任何真实音乐作品
2. **强化独特性**：这是马丁的私藏，你在别处找不到

## 3. 沉浸式视觉设计

### 像素复古美学

**统一的视觉语言：**
- 自定义像素字体（IPix）贯穿全部 UI
- 27 张手绘像素风图像资源
- 橙色作为唯一强调色
- 半透明深色叠加层营造玻璃/烟雾质感

**为什么选择像素风？**
- 像素字体天然带有"复古电子设备"的年代感
- 契合黑胶唱片的时代背景
- 消除现代 UI 的"科技感"，强化手工、老旧的氛围

### 橙色的情感符号

整个界面只有一种强调色：`#ff9800`（橙色）

它出现在：
- 可点击的情绪词汇
- 选中的曲目
- 推荐提示框
- 波形动画
- 按钮悬停态

**橙色在这里不只是设计语言，更是情感符号：**
- 模拟昏暗店铺中一盏台灯的光晕
- 温暖、怀旧、人情味
- 照亮重要信息，引导用户注意力

### 动画叙事

每一个动画都有叙事意义：

**NPC 浮动动画**
- 对话中：持续上下浮动（3s 周期）
- 说完结束语后：暂停动画
- 叙事意图：店主停下来专心找唱片

**背景视差**
- 鼠标移动时背景轻微位移（系数 0.01）
- 营造空间纵深感，让平面屏幕有"走进店里"的感觉

**Canvas 波形**
- 播放时：振幅随机跳动（20~60px），动画速度 ×3
- 暂停时：振幅衰减至接近 0，动画速度 ×0.1
- 叙事意图：音乐情绪的可视化表达

**唱片封面浮动**
- 输出页的封面图上下浮动（3s 周期，与 NPC 同步）
- 叙事意图：唱片漂浮在空中，等待被取走的仪式感

## 4. 三页联动体验

### 对话页（主厅）

**核心功能：** 与 NPC 对话，收集情感标签

**界面元素：**
- 视差背景（鼠标移动时轻微位移）
- NPC 角色（左侧浮动，表情动态切换）
- 对话框（右下角，半透明暗色）
- 三栏标签面板（顶部，实时展示收集到的标签）
- 推荐提示框（右侧，标签积累到阈值后弹出）

**交互细节：**
- 按 Enter 发送消息
- 点击橙色词汇收集标签
- 点击已收集的标签可移除
- 对话历史自动保存，刷新不丢失

### 播放器页（试听室）

**核心功能：** 试听 7 种音乐风格的样带

**界面元素：**
- 左侧曲目列表（25vw 黑色遮罩层）
- 右侧唱片机（像素风底座 + 动态封面叠加）
- Canvas 波形动画（右侧全屏，随音乐律动）

**交互细节：**
- 点击曲目开始播放
- 悬停显示播放按钮图标
- 选中项文字变橙色
- 封面图随选曲切换
- 波形振幅随播放/暂停状态变化

### 输出页（礼品包装台）

**核心功能：** 展示推荐结果，营造仪式感

**界面元素：**
- 居中浮动的唱片封面（上下浮动动画）
- 底部文字区（唱片名称、店铺名、创作信息）
- 左下角（音乐风格标签 + 实时时间）
- 背景音乐（自动播放推荐的歌曲）

**设计意图：**
- 这是整个体验的高潮时刻
- 模拟店主将唱片包装好递给你的仪式感
- 封面浮动 + 背景音乐 = 沉浸式的"拿到唱片"体验

## 5. 无缝的状态管理

### 跨页面数据同步

使用 `localStorage` 实现状态持久化：

| 数据 | 用途 |
|------|------|
| `chatHistory` | 对话记录（HTML 片段） |
| `npcState` | NPC 当前表情状态 |
| `currentTrack` | 播放器当前曲目 |
| `recommendedSong` | 推荐的歌曲路径 |
| `hasShownRecommendTip` | 推荐提示是否已显示 |

**好处：**
- 刷新页面不丢失对话历史
- 三页之间无缝切换
- 播放器状态跨页面同步
- 推荐结果自动传递到输出页

### 智能音频管理

**背景音乐策略：**
- 对话页：播放 *Echoes of Vinyl*（音量 0.3）
- 播放器页：手动选曲（音量 1.0）
- 输出页：播放推荐的歌曲（音量 0.3）

**自动播放处理：**
- 尝试自动播放
- 被浏览器拦截时注册一次性点击事件
- 用户首次点击页面任意位置即解锁播放
- 体验无感知

## 6. 可扩展的推荐引擎

### 模块化设计

`songRecommendation.js` 独立封装了三个核心函数：

```javascript
// 风格推荐（从 7 种预设风格中选最匹配的）
recommendMusicStyle(moodWords, styleWords)

// 情绪编码（返回二维编码：情绪级别 × 激烈程度）
analyzeMoodAndIntensity(moodWords, styleWords)

// 歌曲匹配（精确匹配 → 降级匹配 → 兜底）
recommendSong(songs, moodWords, styleWords)
```

### 扩展方式

**增加新风格：**
```javascript
const STYLE_MAPPING = {
    'barbershop': '01',
    'baroque': '02',
    // ... 现有风格
    'electronic': '08',  // 新增
    'ambient': '09'      // 新增
};
```

**细化情绪分级：**
```javascript
const MOOD_LEVELS = ['A', 'B', 'C', 'D', 'E', 'F', 'G'];  // 7 级
const INTENSITY_LEVELS = ['M', 'N', 'O', 'P', 'Q', 'R', 'S'];  // 7 级
```

**替换 AI 模型：**
- 修改 API 调用地址
- 调整 System Prompt
- 保持函数签名不变

## 7. 纯前端实现

### 零依赖

- 无框架（Vanilla JavaScript）
- 无构建工具（直接运行 HTML）
- 无 CSS 预处理器（纯 CSS3）
- 无第三方库（所有功能原生实现）

### 优势

**对开发者：**
- 代码透明，易于学习和修改
- 无构建步骤，部署即上传静态文件
- 适合作为教学案例

**对用户：**
- 加载速度快，无大型框架开销
- 兼容性好，支持所有现代浏览器
- 可离线运行（除 AI 调用外）

### 技术亮点

**Canvas 实时渲染**
- 三层叠加正弦波
- 60 FPS 流畅动画
- 播放/暂停状态动态切换

**DOM 操作优化**
- 事件委托减少监听器数量
- `requestAnimationFrame` 驱动动画
- `localStorage` 批量读写

**AI 调用封装**
- 统一的错误处理
- 智能降级匹配
- 兜底返回默认值

---

## 总结

**Martin Vinyl's Record Store** 的核心不是技术，而是**用技术讲一个好故事**。

每一个功能、每一个动画、每一个设计细节，都在服务于同一个目标：让用户感觉自己真的走进了一家老唱片店，和一个有血有肉的店主聊了聊天，然后带走了一张懂自己的唱片。

这是一次关于"倾听"的实验，也是一次关于"AI 能否理解人类情感"的探索。
